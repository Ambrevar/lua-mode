    h1. Lua-mode

    table{frame: void}. |Author|immerrr| |Contact| "immerrr+lua@gmail.com":mailto:immerrr+lua@gmail.com|

    lua-mode is an "Emacs":emacs major mode for editing "Lua":lua. The development has moved to "github":lua-mode-github, so update your bookmarks if you still look for new releases at luaforge.

    If you have a problem or a suggestion about lua-mode, please, let me know about it via "issue tracker":lua-mode-tracker.

    h2. INSTALLATION

    h3. MANUAL INSTALLATION

    1. Grab a zipball[1]:
      • "last stable release":https://github.com/immerrr/lua-mode/zipball/rel-20111107
      • "current development snapshot":https://github.com/immerrr/lua-mode/zipball/master

    To install, just copy @lua-mode.el@ into a directory on your @load-path@ (and optionally byte-compile it). To set up Emacs to automatically edit files ending in @.lua@ or with a lua hash-bang line using lua-mode add the following to your init file:

    bc. (autoload 'lua-mode "lua-mode" "Lua editing mode." t) (add-to-list 'auto-mode-alist '("\.lua$" . lua-mode)) (add-to-list 'interpreter-mode-alist '("lua" . lua-mode))

    fn1. or pick any commit you like on "github":lua-mode-github

    h3. USING "EL-GET":el-get

    Installing with el-get is as easy as

    bc. (el-get-install 'lua-mode)

    h2. USAGE

    lua-mode supports c-mode style formatting and sending of lines/regions/files to a Lua interpreter. An interpreter (see variable @lua-default-application@) will be started if you try to send some code and none is running. You can use the process-buffer (named after the application you chose) as if it were an interactive shell. See the documentation for @comint.el@ for details.

    Lua-mode also works with Hide Show minor mode (see @hs-minor-mode@).
